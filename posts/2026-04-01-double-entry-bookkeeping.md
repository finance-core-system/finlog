## 복식부기(Double-Entry Bookkeeping)란?

복식부기는 모든 거래를 두 개의 계정에 동시에 기록하는 회계 방식입니다.
하나의 거래는 항상 **차변(Debit)** 과 **대변(Credit)** 에 동일한 금액이 기록됩니다.

```
차변(Debit)  =  대변(Credit)
```

이 원칙 덕분에 장부는 언제나 균형을 유지합니다.

## 왜 중요한가?

현대 금융 시스템의 모든 원장(Ledger)은 복식부기를 기반으로 설계됩니다.

- **무결성 보장**: 어느 한쪽만 기록되는 오류를 원천 차단
- **감사 추적(Audit Trail)**: 모든 자금 흐름을 양방향으로 추적 가능
- **잔액 검증**: 전체 차변 합계 = 전체 대변 합계가 항상 성립

## 예시: 고객이 10만 원을 입금하는 경우

| 계정 | 차변 | 대변 |
|------|------|------|
| 고객 예수금 | | 100,000 |
| 현금 | 100,000 | |

고객 관점에서는 잔액이 늘었고, 은행 관점에서는 부채(예수금)가 늘었습니다.

## 소프트웨어 원장 설계 시 고려사항

1. **엔트리(Entry)**: 차변/대변 한 쌍을 묶는 단위
2. **트랜잭션 원자성**: 한쪽만 기록되면 데이터 오염 → DB 트랜잭션 필수
3. **이벤트 소싱**: 잔액을 직접 저장하지 않고 모든 엔트리의 합계로 계산하는 설계
